# OdekoDashboards

Static dashboards built from Google Sheets data. Each dashboard is a folder
under `dashboards/`, refreshed hourly by a GitHub Actions workflow.

## Layout

```
.mcp.json                              # registers google-sheets MCP
tools/google-sheets-mcp/               # MCP server (Python, service-account auth)
.github/workflows/refresh-<name>.yml   # one per dashboard
dashboards/<name>/
  index.html                           # Chart.js + vanilla JS, single file
  data.json                            # generated by workflow, committed
  odeko-logo.png                       # copied from any other dashboard
  scripts/build_data.py                # pulls sheet, aggregates, writes data.json
```

## Adding a new dashboard

Model new dashboards on `dashboards/dca1-onhand-eta/`. Steps:

1. Use the `google-sheets` MCP to inspect the source sheet:
   - `get_spreadsheet_info(url)` for tab list
   - `read_range(url, "Tab!A1:Z6")` to see column names and sample data
2. Create `dashboards/<name>/scripts/build_data.py`:
   - Parameterize `SPREADSHEET_ID`, `SHEET_RANGE`, filter values at the top
   - Read `GOOGLE_SERVICE_ACCOUNT_JSON` env var for auth
   - Aggregate by item; output a JSON shaped for the dashboard
3. Create `dashboards/<name>/index.html` (copy DCA1's, adjust charts/columns)
4. Copy `odeko-logo.png` from any existing dashboard folder
5. Create `.github/workflows/refresh-<name>.yml` (copy DCA1's, update name, concurrency group, paths)
6. Confirm the source sheet is shared with `sheets-mcp-bot@sheets-mcp-497414.iam.gserviceaccount.com`
7. Commit + push to main. The push triggers the workflow, which writes `data.json` and commits.

## Workflow conventions

- `git add <file>` BEFORE `git diff --cached --quiet` so new files get detected
- Commit author: `github-actions[bot] <41898282+github-actions[bot]@users.noreply.github.com>`
- Workflow `concurrency.group` is unique per dashboard so they don't block each other

## Secrets / auth

- Service account email: `sheets-mcp-bot@sheets-mcp-497414.iam.gserviceaccount.com`
- Repo secret: `GOOGLE_SERVICE_ACCOUNT_JSON` (already configured) — used by workflows
- Env var: `GOOGLE_SERVICE_ACCOUNT_JSON` (set in Claude Code on the web env) — used by the MCP
- Both hold the same minified service-account JSON key

## Viewing dashboards locally

Browsers block `fetch()` on `file://` URLs, so use a local HTTP server:
```
cd dashboards/<name> && python3 -m http.server 8765
```
Then open http://localhost:8765/.

## Visual style (Odeko brand)

All dashboards must visually match `dashboards/dca1-onhand-eta/index.html`.
Use that file as the design reference for new dashboards.

**Palette (CSS variables, defined in `:root`):**
- `--bg: #FBF7EC` — warm cream background
- `--panel: #FFFFFF` — card surface
- `--line: #E6DFCC` — warm hairline borders
- `--ink: #2A1408` — body text, deep coffee
- `--ink-2: #40180B` — headings, Odeko brand brown
- `--muted: #8A7B6A` — warm gray
- `--accent-yellow: #FFD100` — sparing accent (KPI left border, primary button)
- Status: `--green: #1F7A33`, `--amber: #B5660A`, `--red: #B42318`

**Conventions:**
- Font: Inter (with system fallbacks)
- Sticky header at top with `./odeko-logo.png` and h1 title
- KPI cards in responsive grid; values use `font-variant-numeric: tabular-nums`
- Section cards: white panel, 1px `--line` border, 8px radius, uppercase letter-spaced h3 header
- Chart.js charts: set `Chart.defaults.color = "#8A7B6A"`, `borderColor = "#E6DFCC"`, font family Inter
- Status colors in charts use the palette red / amber / green
- Each dashboard folder ships its own `odeko-logo.png` (copy from another dashboard)
